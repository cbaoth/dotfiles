* [ ] Streamline the sourcing mechanism for commons.sh across scripts. Example:
. Keep one stable loader file (known path).
. Loader finds and sources commons.sh, validates required symbols:
+
[source,bash]
----
# ~/lib/commons-loader.sh: Locate and source commons.sh once
# code: language=bash insertSpaces=true tabSize=2
# -*- mode: sh; sh-shell: bash; indent-tabs-mode: nil; tab-width: 2 -*-
# vim: ft=bash:et:ts=2:sts=2:sw=2
# keywords: bash shell loader commons
# author: Andreas Weyer

cl::require_commons() {
  local -a candidates=(
    "${SCRIPT_PATH}/commons.sh"
    "${SCRIPT_PATH}/lib/commons.sh"
    "${HOME}/lib/commons.sh"
  )
  local f

  for f in "${candidates[@]}"; do
    if [[ -f "${f}" ]]; then
      # shellcheck source=/dev/null
      source "${f}"
      break
    fi
  done

  if ! command -v "cl::cmd_p" > /dev/null 2>&1; then
    printf "commons lib not found, exiting ..\n" >&2
    return 1
  fi

  return 0
}
----
. Scripts use a 1-liner source + 1-liner cl::require_commons call:
+
[source,bash]
----
# ...existing code...
source "${HOME}/lib/commons-loader.sh" || exit 1
cl::require_commons || exit 1
# ...existing code...
----

== 4. Sway / Desktop Setup

=== Replace GDM with lightweight session startup

GDM is currently required only to get a graphical login. Since Sway is started
manually from a TTY (`sudo systemctl stop gdm && sway-nvidia`), GDM adds
overhead with no benefit.

Prerequisite: Sway must be confirmed stable (waybar, keyring, key bindings all
verified working) before making GDM removal permanent.

Options to evaluate (in order of preference):

* *No display manager* — TTY auto-login + auto-start sway from shell profile
** Configure TTY auto-login for tty1 via systemd drop-in:
   `/etc/systemd/system/getty@tty1.service.d/autologin.conf`
** Add to `~/.zprofile`: start `sway-nvidia` automatically if on tty1 and no
   Wayland session is running
** Pros: minimal, no extra packages, full control
** Cons: no graphical greeter (acceptable if YubiKey unlock happens inside sway)

* *greetd + tuigreet* — modern minimal Wayland-native session manager
** `sudo apt install greetd` (Ubuntu 24.04+)
** TUI greeter (`tuigreet`) provides a simple terminal login prompt
** Designed specifically for wlroots/Wayland compositors
** Pros: proper PAM integration (unlocks login keyring), session selection
** Cons: newer, fewer tutorials for Ubuntu

* *LightDM* — familiar from previous use
** Well-supported on Ubuntu, handles PAM/keyring correctly
** Slick-greeter or gtk-greeter available
** More overhead than greetd/no-DM; X11-centric but Wayland sessions work
